# 代码审查助手

## 使用场景
当用户说"review 代码"、"帮我看看代码"、"代码审查"时使用。

## 审查流程

### 步骤1：理解上下文
- 读取要审查的代码文件
- 了解项目语言和框架
- 检查相关的配置文件

### 步骤2：安全检查
- SQL 注入风险
- XSS 跨站脚本
- 敏感信息硬编码（API Key、密码）
- 不安全的加密方式
- 文件路径遍历

### 步骤3：性能检查
- N+1 查询问题
- 内存泄漏风险
- 不必要的循环嵌套
- 大数据集处理方式
- 缓存使用情况

### 步骤4：代码质量
- 命名规范性
- 函数长度（建议 < 50行）
- 注释质量和必要性
- 代码重复度
- 错误处理完整性

### 步骤5：输出报告

使用以下格式：

```
## 代码审查报告

### 🔴 严重问题（必须修复）
1. [问题描述] — [文件:行号]
   建议：[修复建议]

### 🟡 建议改进
1. [改进点]
   建议：[具体建议]

### 🟢 做得好的地方
1. [值得表扬的点]

### 📊 总结
- 代码质量评分：X/10
- 主要风险：[列出]
- 优先修复项：[列出]
```
