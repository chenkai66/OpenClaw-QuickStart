# 第7章·第5节：内容研究与写作流水线

> 从选题到发布，让 AI 帮你完成内容创作中最耗时的部分：调研和初稿。

---

## 场景

你是一个技术博主、自媒体作者或市场团队成员。每周需要产出 2-3 篇文章，但 70% 的时间都花在了找资料和写初稿上。

OpenClaw 可以帮你把这个过程自动化：

```mermaid
graph LR
    Topic[选题灵感] --> Research[深度调研]
    Research --> Outline[生成大纲]
    Outline --> Draft[撰写初稿]
    Draft --> Review[人工审核]
    Review --> Publish[排版发布]
```

> 核心原则：AI 做调研和初稿，**人做审核和发布**。永远不要自动发布未经审核的内容。

---

## 工作流一：深度调研 Agent

这是社区中使用频率极高的一个模式——并行调研多个信息源：

```
你：帮我深度调研一下 "2026年 AI Agent 框架对比：OpenClaw vs AutoGPT vs CrewAI"

Agent 的工作方式：
1. 并行搜索 Twitter、Reddit、HackerNews、GitHub、博客
2. 收集各框架的：Stars 数、最近更新、社区活跃度、真实用户评价
3. 提取：共同优点、各自特色、常见吐槽
4. 整理成结构化报告
```

### 调研提示词模板

```
我需要深度调研 [主题]。请按以下方式进行：

分别搜索以下信息源：
1. Twitter/X — 最近 2 周的讨论和评价
2. Reddit — 相关 subreddit 的帖子和评论
3. Hacker News — 相关故事和评论区讨论
4. GitHub — 星标数、最近提交活跃度、Issue 数量
5. 博客/文章 — 最新的评测和教程

对每个信息源，整理：
- 关键发现和洞察
- 正面和负面评价
- 具体数据（星标数、用户量等）
- 值得引用的原话

最后综合所有信息，输出：
1. 执行摘要（一段话总结现状）
2. 关键对比表格
3. 常见观点和争议点
4. 没人讨论但值得关注的角度（选题机会）
5. 所有来源链接

保存完整报告到 ~/obsidian/Research/ 目录。
```

## 工作流二：自动选题与内容日历

每周一早上自动分析热点，推荐本周选题：

```bash
openclaw cron add \
  --name "Weekly Topics" \
  --cron "0 9 * * 1" \
  --tz "Asia/Shanghai" \
  --session isolated \
  --message "本周内容选题分析：

1. 搜索过去 7 天 AI/开发工具领域的热门话题
2. 分析 GitHub Trending 本周热门项目
3. 检查我之前的文章列表（~/obsidian/Published/），避免重复
4. 结合热度和我的专长领域（AI Agent、开发效率、开源工具），推荐 5 个选题
5. 每个选题给出：标题建议、预估热度、推荐角度、参考资料

输出内容日历格式：
| 发布日 | 选题 | 角度 | 参考 |
|-------|------|------|------|

保存到 ~/obsidian/Calendar/$(date +%Y)-W$(date +%V).md"
```

## 工作流三：从调研到初稿

当你确定了选题后：

```
你：就写"OpenClaw vs AutoGPT"这个话题，用调研报告里的数据，
    写一篇面向开发者的对比文章。2000-3000 字。

Agent：
（使用之前的调研报告 + 补充搜索最新数据）

📝 初稿已生成：~/obsidian/Drafts/openclaw-vs-autogpt.md

结构：
1. 引言：为什么2026年需要 AI Agent 框架
2. 快速对比表（Stars/语言/生态/学习曲线）
3. 架构差异（OpenClaw 单进程 vs AutoGPT 多步骤）
4. 实际体验对比（安装/配置/日常使用）
5. 社区生态对比
6. 选择建议：什么场景用什么工具
7. 结论

需要我调整哪个部分？
```

### 写作风格配置

在 SOUL.md 里定义你的写作风格：

```markdown
## 写作风格
- 语言风格：口语化技术写作，像在跟朋友解释
- 避免：学术腔、过度营销、空洞的赞美
- 偏好：用具体数据说话，给真实例子，承认缺点
- 结构：每 3-4 段加一个小标题，代码块用简短的例子
- 长度：控制在 2000-3000 字，太长就拆成系列
```

## 工作流四：URL 摘要（内置技能）

日常效率最高的小工具——把任何链接变成结构化摘要：

```
你：/summarize https://blog.example.com/some-long-article

Agent：
📄 文章摘要：《如何搭建生产级 AI Agent》

核心观点：
1. 不要从零开始，用 OpenClaw 或 LangChain 等框架
2. 关键是记忆管理，不是模型选择
3. 生产环境必须有监控和降级策略

关键数据：
- 作者测试了 5 个框架，OpenClaw 评分最高（8.5/10）
- 平均部署时间：2 小时（有经验）到 2 天（新手）

值得注意：
- 作者是 OpenClaw 贡献者，可能有偏见
- 文章发布于 2026年3月，数据比较新
```

这个技能不需要额外配置，安装时默认就有。每天用 3-5 次，是社区中使用频率最高的单个功能。

---

## 注意事项

1. **永远不要自动发布**：AI 初稿需要人工审核，检查事实准确性
2. **标注 AI 辅助**：如果你的平台要求标注 AI 参与，请遵守
3. **版权意识**：调研时注意信息来源，引用要注明出处
4. **避免同质化**：AI 初稿是起点，你的独特观点和经验才是价值所在
