# 第7章·第3节：每日简报与健康监控

> 每天早上醒来，一条消息搞定所有你需要知道的事。社区投票 #1 最实用的 OpenClaw 应用。

---

## 为什么这是最火的用例？

社区里几乎每个 OpenClaw 用户都会搭的第一个自动化就是**每日简报**。原因很简单：

- 搭建只要 30 分钟
- 每天节省 10-15 分钟（翻 5 个 App 的时间）
- **认知收益巨大**——起床后看一条消息就知道今天的重点

---

## 方案一：基础每日简报（30 分钟搭建）

### 效果

每天早上 7:30，在钉钉/Telegram/Discord 收到一条消息：

```
🌅 2026年3月25日 周二 简报

📅 今日日程（3 件）
- 10:00 产品周会（腾讯会议，30分钟）
- 14:00 和小王1v1（准备：上次答应帮他看代码）
- 16:00 客户演示（注意：客户对延迟问题敏感）

🌤️ 天气
北京：晴 8-18°C，适合出门

📧 重要邮件（2 封）
- [紧急] AWS 账单异常：本月已用 $230（预算 $200）
- 产品经理发了新需求文档，标记了你

📰 科技要闻
- OpenClaw v2026.3.14 发布：新增 MCP 批量调用
- Anthropic 宣布 Claude 5 将支持本地运行

💡 今日建议
重点关注客户演示，建议提前 30 分钟准备延迟优化的数据。
```

### 配置

```bash
openclaw cron add \
  --name "Morning Briefing" \
  --cron "30 7 * * *" \
  --tz "Asia/Shanghai" \
  --delivery announce \
  --message "生成今日简报，包含以下内容：

1. 📅 今日日程：查看 Google Calendar 中今天的所有事件，对每个会议给出准备建议
2. 🌤️ 天气：搜索我所在城市（北京）的今日天气
3. 📧 重要邮件：检查过去 12 小时的新邮件，只列出需要我关注的（忽略营销邮件和通知）
4. 📰 科技要闻：搜索 AI、开发工具领域的最新新闻（2-3 条即可）
5. 💡 今日建议：基于以上信息，给出 1 条今天最需要关注的事

格式要求：简洁，用 emoji 分区，整体控制在手机一屏能看完的长度。"
```

## 方案二：带健康监控的增强简报

除了早晨简报，社区里很多人还会加上**全天候健康监控**——每 30 分钟静默检查一次，**只在出问题时才通知你**。

### 健康监控 Cron

```bash
openclaw cron add \
  --name "Health Check" \
  --cron "*/30 7-23 * * *" \
  --tz "Asia/Shanghai" \
  --session isolated \
  --message "静默健康检查（只在需要时通知我）：

1. 邮件：检查最近 30 分钟的新邮件，只标记紧急的（付款失败、安全告警、会议变更）
2. 日历：检查未来 2 小时的日程，只提醒我还没准备的会议
3. 服务器（如果配置了 SSH）：检查 CPU/内存/磁盘，只在异常时告警

规则：
- 没有异常就不要发消息，不需要 '一切正常' 的确认
- 严重程度分两级：'紧急'（1小时内需处理）和 '提醒'（今天内关注）
- 邮件只读不发，可以起草回复但绝不代我发送"
```

### 运行效果

大部分时间你不会收到任何消息（说明一切正常）。但当出现问题时：

```
⚠️ 紧急 | 14:32
AWS CloudWatch 告警邮件：us-east-1 的 API 延迟超过 2s。
建议：检查最近的部署，可能是 15 分钟前的代码发布导致。

💡 提醒 | 15:01
16:00 的客户演示还有 1 小时。你上次说要准备延迟优化数据，
我搜了一下最近 7 天的 API 平均延迟是 180ms（正常范围），
需要我整理成图表吗？
```

## 方案三：Twitter/社交媒体晨报

适合需要追踪行业动态的用户：

```bash
openclaw cron add \
  --name "Social Briefing" \
  --cron "0 8 * * *" \
  --tz "Asia/Shanghai" \
  --session isolated \
  --message "扫描以下信息源，生成今日科技社交简报：

1. 用 web_search 搜索过去 24 小时 AI/开发工具领域的热门话题
2. 搜索 Hacker News 首页的热门讨论
3. 搜索 GitHub Trending（今日）

输出格式：
- 🔥 热门话题（3-5 个，每个一句话摘要 + 链接）
- 💬 值得关注的讨论（1-2 个有深度的社区讨论）
- 🚀 GitHub 热门项目（2-3 个今日新上榜的项目）
- 📝 视频/文章灵感（如果有话题适合写文章或做视频，记录到笔记）

保存完整版到 ~/obsidian/Daily/$(date +%Y-%m-%d)-social.md
聊天中只发精简版。"
```

## 进阶配置

### 根据星期调整简报内容

```bash
# AGENTS.md 中加入
## 简报规则
- 周一简报：增加"本周目标回顾"
- 周五简报：增加"本周总结 + 下周规划建议"
- 周末简报：减少工作内容，增加天气和生活建议
```

### 多渠道投递

简报可以同时发到多个渠道：

| 渠道 | 适合 | 配置 |
|------|------|------|
| 钉钉 | 工作日程 | 默认渠道 |
| Telegram | 个人简报 | `--delivery announce` |
| Discord | 团队共享 | 指定频道 |
| 邮件 | 归档留存 | 通过 SMTP 工具 |
