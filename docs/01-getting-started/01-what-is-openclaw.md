# 第1章·第1节：OpenClaw 是什么？

> 一句话：OpenClaw 是你自己的 AI 助手，跑在你自己的机器上，能帮你发消息、管邮件、写代码、跑脚本、操控浏览器。

---

## 简单说

**OpenClaw 是一个开源的、可自托管的 AI Agent 网关平台。**

它把你常用的聊天工具（微信、钉钉、飞书、企业微信、QQ、Telegram、Discord、Slack、WhatsApp、Signal、iMessage、Google Chat、Teams、Matrix 等 20 多个平台）统一接入 AI 代理，让你在任何地方都能指挥 AI 干活。

2026 年 3 月，OpenClaw 的 GitHub Stars 突破 **25.1 万**，超越 React（24.3 万）和 Linux（22 万），登顶 GitHub 历史星标第一。最新稳定版 **v2026.3.13**。

---

## 跟 ChatGPT / Claude 有什么区别？

| 对比项 | ChatGPT / Claude | OpenClaw |
|--------|------------------|----------|
| 部署方式 | 厂商云端 | **自己部署**，数据完全在你手里 |
| 能力范围 | 聊天、回答问题 | **执行任务**：发邮件、跑命令、操作文件 |
| 使用入口 | 只能网页或 App | **任意聊天平台**：钉钉、微信、Telegram、Discord 等 |
| 扩展性 | 有限 | **Skills 系统**：ClawHub 上有大量社区技能 |
| 记忆 | 有限 | **持久记忆**：用得越久越了解你 |
| 定时任务 | 不支持 | **Cron 系统**：每天早上 7 点自动发日报 |
| 上下文管理 | 固定策略 | **ContextEngine 可插拔引擎**：自由切换上下文策略 |
| 开源 | 否 | **MIT 协议**，随便用 |

## OpenClaw 能干什么？（真实场景）

### 日常生活
- "查一下明天北京天气，发到我 Telegram"
- "每天早上 7 点给我发今日待办"
- "帮我用浏览器比一下这个耳机在淘宝和京东的价格"

- "把未读邮件按优先级整理一下"

- "Review 一下这个 PR，给出代码审查意见"

- "在微信里直接和 AI 对话，不用开电脑"
- "帮我总结桌面上的会议记录，结果发到微信"
- "微信群里 @AI 自动回答技术问题"

## 核心架构（6 层）

| 层级 | 作用 |
|------|------|
| **Gateway 网关** | 中央枢纽——消息路由、会话管理、插件、工具执行 |
| **Channels 渠道** | 把钉钉 / 微信 / Telegram / Discord 等消息统一转成标准格式 |
| **路由 + 会话** | 决定哪个 Agent 处理哪个对话 |
| **Agent 运行时** | 处理上下文、调用模型、流式响应、请求工具 |
| **ContextEngine** | 可插拔上下文引擎——管理记忆、压缩、RAG 检索 |
| **Tools + Skills** | 能力层——26 个 Tools + 社区 Skills + MCP 协议扩展 |

---

## 设计理念

1. **自托管优先** — 你的数据你做主
2. **多渠道统一** — 一个网关服务所有聊天平台
3. **Agent 原生** — 天生为 AI Agent 设计，带工具调用、会话、记忆
4. **可插拔架构** — ContextEngine、渠道、Skills 全部可替换
5. **开放社区** — MIT 协议，社区驱动，ClawHub 上有大量可用技能
