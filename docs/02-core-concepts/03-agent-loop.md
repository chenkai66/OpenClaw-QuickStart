# 第2章·第3节：Agent 循环

> 了解 AI Agent 是如何"思考-行动-观察"的。

---

## Agentic Loop（代理循环）

OpenClaw 的 AI Agent 运行在一个持续的循环中：

```
        ┌─────────┐
        │ 感知    │ ← 接收用户消息或触发事件
        │Perceive │
        └────┬────┘
             │
        ┌────▼────┐
        │ 规划    │ ← LLM 分析意图，制定计划
        │  Plan   │
        └────┬────┘
             │
        ┌────▼────┐
        │ 执行    │ ← 调用 Tools 完成任务
        │  Act    │
        └────┬────┘
             │
        ┌────▼────┐
        │ 观察    │ ← 检查执行结果
        │Observe  │
        └────┬────┘
             │
        ┌────▼────┐
        │ 沟通    │ ← 向用户报告结果
        │Communicate│
        └────┬────┘
             │
             └──→ 等待下一个输入...
```

---

## 实际执行示例

**用户发送**：`帮我把 /tmp/data.csv 按第三列排序，保存为 /tmp/sorted.csv`

**Agent 的思考过程**：

```
1. [感知] 收到文件排序请求
2. [规划] 计划：
   - 先读取文件了解结构
   - 使用 sort 命令按第三列排序
   - 保存结果
   - 验证输出
3. [执行] 调用 read 工具读取 data.csv 前几行
4. [观察] 文件是 CSV 格式，第三列是数字
5. [执行] 调用 exec 工具：sort -t',' -k3 -n /tmp/data.csv > /tmp/sorted.csv
6. [观察] 命令执行成功，exit code 0
7. [执行] 调用 read 工具验证 sorted.csv
8. [观察] 数据已正确排序
9. [沟通] "已完成排序！sorted.csv 共 1000 行，按第三列数字升序排列。"
```

## 触发方式

Agent 循环可以被多种事件触发：

| 触发方式 | 示例 |
|----------|------|
| 用户消息 | 在钉钉发送消息 |
| Cron 定时 | 每天 7:00 触发 |
| 文件变化 | 监控文件夹变化 |
| Webhook | 外部系统调用 |
| 其他 Agent | 多 Agent 协作 |
