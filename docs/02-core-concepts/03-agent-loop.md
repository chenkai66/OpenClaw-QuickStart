# 第2章·第3节：Agent 循环

> 了解 AI Agent 是如何"思考-行动-观察"的。

---

## Agentic Loop（代理循环）

OpenClaw 的 AI Agent 运行在一个持续的循环中：

```
        ┌─────────┐
        │ 感知    │ ← 接收用户消息或触发事件
        │Perceive │
        └────┬────┘
             │
        ┌────▼────┐
        │ 规划    │ ← LLM 分析意图，制定计划
        │  Plan   │
        └────┬────┘
             │
        ┌────▼────┐
        │ 执行    │ ← 调用 Tools 完成任务
        │  Act    │
        └────┬────┘
             │
        ┌────▼────┐
        │ 观察    │ ← 检查执行结果
        │Observe  │
        └────┬────┘
             │
        ┌────▼────┐
        │ 沟通    │ ← 向用户报告结果
        │Communicate│
        └────┬────┘
             │
             └──→ 等待下一个输入...
```

## 实际执行示例

**用户发送**：`帮我把 /tmp/data.csv 按第三列排序，保存为 /tmp/sorted.csv`

**Agent 的思考过程**：

```
1. [感知] 收到文件排序请求
2. [规划] 计划：
   - 先读取文件了解结构
   - 使用 sort 命令按第三列排序
   - 保存结果
   - 验证输出
3. [执行] 调用 read 工具读取 data.csv 前几行
4. [观察] 文件是 CSV 格式，第三列是数字
5. [执行] 调用 exec 工具：sort -t',' -k3 -n /tmp/data.csv > /tmp/sorted.csv
6. [观察] 命令执行成功，exit code 0
7. [执行] 调用 read 工具验证 sorted.csv
8. [观察] 数据已正确排序
9. [沟通] "已完成排序！sorted.csv 共 1000 行，按第三列数字升序排列。"
```

Agent 循环可以被多种事件触发：

| 触发方式 | 示例 |
|----------|------|

## Lobster Loop（龙虾循环）

OpenClaw 的 Agent 核心运行机制叫 **Lobster Loop**——一种"思考-行动"循环：

```
1. 接收消息（用户/系统/定时）
2. 加载上下文（SOUL + USER + 记忆）
3. 思考（选择策略）
4. 行动（调用工具/生成回复）
5. 观察结果
6. 如果任务未完成 → 回到 3
7. 返回最终结果
```

关键特点：
- **自主决策**：AI 自己决定调用哪些工具、分几步完成
- **工具编排**：一次对话可能调用多个工具（搜索 → 读文件 → 写代码 → 执行）
- **错误恢复**：工具调用失败时，AI 会自动调整策略重试

## 思考级别（Thinking Levels）

OpenClaw 支持控制 AI 的"思考深度"：

```bash
# 默认（不额外思考）
openclaw agent --message "今天天气怎么样"

# 高级思考（复杂问题）
openclaw agent --message "分析这段代码的安全性" --thinking high
```

> 思考级别越高，AI 的推理越详细，但耗时和 Token 消耗也越多。
